# Overview  
**Dedi** is a self-hosted, privacy-first rebrand of Twake Chat that runs on our own domain **`*.dedim.com.tr`**. It replaces web-based phone/OTP login with **native Flutter screens**, persists the user's Matrix session for seamless re-entry, and swaps ToM-server's **Octopush** SMS dependency for a **multi-provider** SMS layer (**Turkcell MesajÜssü v2**, **NetGSM**, **İletiMerkezi**, **TurkTelekom**, optional **Twilio**).  
It's built for teams who want Matrix-based messaging with first-class phone OTP onboarding while **owning** the infra, domains, and SMS routing.

# Technical Architecture  
- **System Components**
  - **Flutter Client (Dedi)**: Replaces WebView SSO with native OTP screens; centralizes endpoints; persists Matrix session.  
  - **ToM-server Monorepo**: Contains **@twake/matrix-identity-server** and `packages/matrix-invite/`. We **remove Octopush** and route SMS via `MultiProvider`.  
  - **Matrix Homeserver**: Synapse at `matrix.dedim.com.tr`.  
  - **Reverse Proxy**: Nginx-proxy handling TLS/host routing.  
  - **Optional Push**: ntfy at `notification.dedim.com.tr`.

- **Data Models**
  - **Client Flags**: `first_time: bool` (SharedPreferences), `logged_in: bool`.  
  - **Matrix Session**: Stored by Matrix SDK (Hive/Isar).  
  - **(If thin adapter used)** transient OTP req/verify payloads only; no new server-side OTP store (reuse identity v2).

- **APIs & Integrations**
  - **Identity v2 / Invite** (reuse-first): request/submit/bind phone tokens; `store-invite` for `msisdn`.  
  - **Multi-SMS Provider (Node.js)**
    - Provider interface:
      ```ts
      interface SMSProvider {
        readonly name: string;
        sendSMS(args:{to:string;text:string;sender?:string;campaignName?:string}):Promise<{ok:boolean;provider:string;id?:string;raw?:any}>;
        validateConfig?():void;
      }
      ```
    - **Turkcell MesajÜssü v2**
      - Login: `POST ${TURKCELL_BASE_URL}/integration/v2/login` `{ username, password }` → `{ token, tokenExpireDate, sessionInactivityPeriod }` (cache/refresh).  
      - Send: `POST ${TURKCELL_BASE_URL}/integration/v2/sms/job/json` headers: `token: <token>`; body:
        `{"messageText":"...","sender":"<SENDER>","campaignName":"OTP","receivers":["5XXXXXXXXX"]}`  
        Convert `+905XXXXXXXXX` → `5XXXXXXXXX`.
    - **NetGSM**:  
      GET `https://api.netgsm.com.tr/sms/send/get?usercode=<U>&password=<P>&gsmno=<MSISDN>&message=<TEXT>&msgheader=<SENDER>`  
      `"00*"` → success.
    - **İletiMerkezi**:  
      GET `https://api.iletimerkezi.com/v1/send-sms/get/?key=<KEY>&hash=<HASH>&text=<TEXT>&receipents=<MSISDN>&sender=<TITLE>`  
      `status==="success"` → success.
    - **TurkTelekom**: scaffold with TODOs (env keys in place).  
    - **Twilio**: optional standard REST.
    - **Env & Fallback**
      - Either `SMS_PROVIDERS=turkcell,netgsm,iletimerkezi,twilio` **or**:
        `MAIN_SMS_PROVIDER=turkcell` + `FALLBACK_SMS1=netgsm` + `FALLBACK_SMS2=iletimerkezi` + `FALLBACK_SMS3=twilio`  
      - Fallback only on transient errors (timeout/network/5xx); **no** fallback on 4xx/auth/validation.

- **Infrastructure Requirements**
  - Domains: `matrix|identity|chat|notification.dedim.com.tr`.  
  - Docker: synapse, identity-server, (optional) msisdn-auth thin adapter, ntfy, redis (if already used elsewhere).  
  - CI guard: fail builds when non-`dedim.com.tr` hosts present in client/server.

# Risks and Mitigations  
- **Provider Protocol Variance**  
  - *Risk*: Different HTTP methods, response codes, phone formats.  
  - *Mitigation*: Strict provider adapters; normalize to `{ok,provider,id?,raw}`; convert E.164 → provider format; comprehensive unit tests.

- **Turkcell Token Lifecycle**  
  - *Risk*: Token expiry/inactivity causing hidden failures.  
  - *Mitigation*: Cache with expiry; auto-refresh on 401; circuit-breaker metrics.

- **Accidental OTP Re-Implementation**  
  - *Risk*: Duplicating identity v2 logic.  
  - *Mitigation*: **Reuse-first** mandate; thin adapter only if payloads need normalization.

- **Domain Drift**  
  - *Risk*: Missed URLs causing mixed-content or auth failures.  
  - *Mitigation*: CI grep blocking non-`dedim.com.tr`; single Environment config.

- **UX Drift from Designs**  
  - *Risk*: Screens diverge from `/login_flow`.  
  - *Mitigation*: Pixel-checklist; design review gate; snapshot tests.
